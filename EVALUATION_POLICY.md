# 評価の方針

本書では，swallow-evaluation-instruct（以下「本フレームワーク」）の設計およびモデル評価にあたってSwallowチームが採用している基本方針を紹介します．例外はありますが，本フレームワークは原則として本方針に沿って開発しています．

## 事後学習済みモデルを優先する

Swallow チームは[swallow-evaluation](https://github.com/swallow-llm/swallow-evaluation)を開発し，これまでは「文の続きを予測する」ことに長けた事前学習済みモデル向けの評価方法を採用してきました．具体的には，例示によってタスクを説明するFew-shot推論や，停止条件を設定しやすいよう回答のみを出力させるプロンプト設計などです．百科事典的知識や常識に基づく質問応答といった知識依存型タスク，機械翻訳や自動要約といった従来の自然言語処理タスクにおいては，これらの評価方法で事前学習の効果を適切に測定できることを確認しています．
一方で，強化学習などを通じてツール使用能力や深い推論能力を獲得するなど，事後学習（ファインチューニング，強化学習など）の役割は，従来の有用性や安全性の学習（いわゆるアライメント）にとどまらず，モデルの実用性や問題解決能力に大きく影響するようになってきました．そこで本フレームワークは「考えてから解く」ふるまいを前提とする事後学習済みモデルの評価を重視して開発しています．具体的には，推論過程も含めて白紙から考えるZero-shot推論，推論を促す思考の連鎖（Chain-of-Thought）プロンプトの導入，短答ではなく推論過程を含む自由生成からの回答抽出などの方法を採用しています．

## 文生成条件は実行時に指定する

近年の事後学習済みモデルでは，推論過程を延ばす，複数候補を生成して選択するなど，学習時ではなく推論時の計算予算を増やすことで効率よくモデルの性能を向上できる「推論時のスケール則」が報告されています（[Snell et al., 2025](https://openreview.net/forum?id=4FWAwZtd2n)）．このような効果を検証するには多様な文生成条件でモデルを評価する必要があります．またSwallowチームの経験では影響は概して限定的であるものの，深い推論を安定させるために特定のtemperatureやtop-pなどのサンプリングパラメータを推奨するモデルが増えつつあります．
これらの研究および実務における傾向をふまえて，本フレームワークでは，サンプリングパラメータや最大出力トークン長などの生成条件を原則として評価実行時の引数で指定できるようにして，評価の目的や評価するモデル・ベンチマークの特性に応じてエンドユーザが調整できる設計としています．
なお[Swallow LLM Leaderboard v2](https://swallow-llm.github.io/swallow-leaderboard-v2.ja.html) では，2025年8月時点では大半のベンチマークで貪欲法（temperature=0）による決定的デコーディングを採用しています（参考：[Swallow チームが実装したベンチマーク一覧](./BENCHMARKS.md)）．一方で，複数回試行によるスケール効果の定量化に適した Pass@K 指標（K回試行のうち少なくとも1回が正解となる確率）の導入を進めており，将来的には確率的デコーディングによる評価を主体にすることを検討しています．

## 推論型モデルをサポートする

OpenAI o系列やDeepSeek-R1に代表されるような，推論過程とユーザへの最終回答をマークアップによって明示的に分離するタイプを[推論型モデル](https://openai.com/ja-JP/index/introducing-openai-o1-preview/)は事後学習モデルの主流となりつつあります．本フレームワークでは推論型モデルを適切に評価するために，推論過程と最終回答の分離，推論の有無や深さの制御に対応しています．vLLMによってモデルの出力を推論過程と最終回答に分離したうえで，モデルの出力から推論過程を除去した最終回答を評価対象とするが，最終回答がnullになる場合は出力を最終回答として扱う仕様にしています．これは推論の要否を自律的に判断するモデルが将来的に出現することを想定したものです．推論の有無や深さの制御については，いわゆる reasoning effortおよび "thinking mode on" のようなシステムメッセージを評価実行時の引数で指定できるようにしています．

OpenAI の o系列や DeepSeek-R1 に代表される，推論過程とユーザーへの最終回答を明示的に分離する「推論型モデル」は，事後学習モデルの主流になりつつあります（参考：[OpenAI o1](https://openai.com/ja-JP/index/introducing-openai-o1-preview/)）．本フレームワークでは推論型モデルを適切に評価するため，推論過程と最終回答の分離，および推論の有無や深さの制御に対応しています。
前者については，主にvLLMを用いてモデル出力を推論過程と最終回答に分離して，最終回答のみを評価対象としています．ただし推論の要否を自律的に判断するモデルの出現を見据えて，最終回答が null となる場合は、出力全体を最終回答として扱う仕様です。
推論の有無や深さの制御については、いわゆる reasoning effort や “thinking mode on” といったシステムメッセージを、評価実行時の引数として指定できるようにしています。
